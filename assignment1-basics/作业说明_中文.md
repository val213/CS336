## 2 Byte-Pair Encoding（BPE）分词器

在作业的第一部分，我们将训练并实现一个字节级 byte-pair encoding（BPE）分词器（Sennrich et al., 2016; Wang et al., 2019）。具体来说，我们将任意（Unicode）字符串表示为字节序列，并在这个字节序列上训练 BPE 分词器。之后，我们将用这个分词器将文本（字符串）编码为 token（整数序列），用于语言建模。

### 2.1 Unicode 标准

Unicode 是一种文本编码标准，将字符映射到整数码点（code point）。截至 Unicode 16.0（2024年9月发布），该标准在168个书写系统中定义了154,998个字符。例如，字符 "s" 的码点是115（通常记作 U+0073，其中 U+ 是惯用前缀，0073 是115的十六进制表示），字符 "牛" 的码点是29275。在 Python 中，可以用 `ord()` 函数将单个 Unicode 字符转换为其整数表示，`chr()` 函数则将整数 Unicode 码点转换为包含对应字符的字符串。

```python
>>> ord('牛')
29275
>>> chr(29275)
'牛'
```

### 2.2 Unicode 编码

虽然 Unicode 标准定义了字符到码点（整数）的映射，但直接在 Unicode 码点上训练分词器并不实用，因为词表会过于庞大（约15万个条目）且稀疏（许多字符非常罕见）。因此，我们将使用 Unicode 编码，将 Unicode 字符转换为字节序列。Unicode 标准本身定义了三种编码：UTF-8、UTF-16 和 UTF-32，其中 UTF-8 是互联网上最主流的编码（超过98%的网页使用）。

要将 Unicode 字符串编码为 UTF-8，可以使用 Python 的 `encode()` 函数。要访问 Python `bytes` 对象的底层字节值，可以对其进行迭代（例如调用 `list()`）。最后，可以用 `decode()` 函数将 UTF-8 字节字符串解码为 Unicode 字符串。

```python
>>> test_string = "hello! こんにちは!"
>>> utf8_encoded = test_string.encode("utf-8")
>>> print(utf8_encoded)
b'hello! \xe3\x81\x93\xe3\x82\x93\xe3\x81\xab\xe3\x81\xa1\xe3\x81\xaf!'
>>> print(type(utf8_encoded))
<class 'bytes'>
>>> # 获取编码字符串的字节值（0到255的整数）
>>> list(utf8_encoded)
[104, 101, 108, 108, 111, 33, 32, 227, 129, 147, 227, 130, 147, 227, 129, 171, 227, 129,
161, 227, 129, 175, 33]
>>> # 一个字节不一定对应一个 Unicode 字符！
>>> print(len(test_string))
13
>>> print(len(utf8_encoded))
23
>>> print(utf8_encoded.decode("utf-8"))
hello! こんにちは!
```

通过将 Unicode 码点转换为字节序列（例如通过 UTF-8 编码），我们实际上是将码点序列（范围0到154,997的整数）转换为字节值序列（范围0到255的整数）。256长度的字节词表更易于处理。使用字节级分词时，不需要担心词表外（OOV）的 token，因为任何输入文本都可以表示为0到255的整数序列。

### 2.3 子词分词（Subword Tokenization）

虽然字节级分词可以解决词级分词器面临的 OOV 问题，但将文本分词为字节会产生极长的输入序列。这会减慢模型训练速度，因为10个词的句子在词级语言模型中可能只有10个 token，但在字符级模型中可能有50个或更多 token（取决于词的长度）。处理这些更长的序列在模型的每一步都需要更多计算。此外，在字节序列上进行语言建模很困难，因为更长的输入序列会在数据中产生更长的长程依赖。

子词分词是词级分词器和字节级分词器之间的折中方案。字节级分词器的词表有256个条目（字节值范围0到255）。子词分词器以更大的词表规模换取更好的输入字节序列压缩比。例如，如果字节序列 `b'the'` 在训练数据中频繁出现，将其作为词表条目可以将这个3-token序列减少为单个 token。

如何选择这些子词单元加入词表？Sennrich et al.（2016）提出使用 byte-pair encoding（BPE；Gage, 1994），这是一种压缩算法，迭代地将最频繁的字节对"合并"为一个新的未使用索引。注意，这个算法通过最大化输入序列的压缩来向词表添加子词 token——如果一个词在输入文本中出现次数足够多，它就会被表示为单个子词单元。

用 BPE 构建词表的子词分词器通常被称为 BPE 分词器。在本作业中，我们将实现一个字节级 BPE 分词器，其中词表项是字节或合并后的字节序列，在 OOV 处理和可管理的输入序列长度方面兼顾两者的优点。构建 BPE 分词器词表的过程称为"训练"BPE 分词器。

### 2.4 BPE 分词器训练

BPE 分词器训练过程由三个主要步骤组成。

**词表初始化**
分词器词表是从字节字符串 token 到整数 ID 的一一映射。由于我们训练字节级 BPE 分词器，初始词表就是所有字节的集合。由于有256个可能的字节值，初始词表大小为256。

**预分词（Pre-tokenization）**
有了词表后，原则上可以统计字节在文本中彼此相邻出现的频率，并从最频繁的字节对开始合并。然而，这在计算上非常昂贵，因为每次合并都需要对语料库进行完整遍历。此外，直接跨语料库合并字节可能会产生仅在标点符号上不同的 token（例如 `dog!` 和 `dog.`），这些 token 会获得完全不同的 token ID，尽管它们在语义上高度相似（仅标点不同）。

为避免这种情况，我们对语料库进行预分词。你可以将其视为对语料库的粗粒度分词，帮助我们统计字符对的出现频率。例如，词 `'text'` 可能作为预分词单元出现10次。在这种情况下，当我们统计字符 `'t'` 和 `'e'` 相邻出现的频率时，我们会发现词 `'text'` 中 `'t'` 和 `'e'` 相邻，可以将计数增加10，而不必遍历整个语料库。由于我们训练字节级 BPE 模型，每个预分词单元表示为 UTF-8 字节序列。

Sennrich et al.（2016）的原始 BPE 实现通过简单地按空格分割来预分词（即 `s.split(" ")`）。相比之下，我们将使用基于正则表达式的预分词器（GPT-2 使用；Radford et al., 2019），来自：

```
https://github.com/openai/tiktoken/pull/234/files
```

```python
>>> PAT = r"""'(?:[sdmt]|ll|ve|re)| ?\p{L}+| ?\p{N}+| ?[^\s\p{L}\p{N}]+|\s+(?!\S)|\s+"""
```

可以用这个预分词器交互式地分割一些文本，以更好地理解其行为：

```python
>>> # 需要 `regex` 包
>>> import regex as re
>>> re.findall(PAT, "some text that i'll pre-tokenize")
['some', ' text', ' that', ' i', "'ll", ' pre', '-', 'tokenize']
```

但在代码中，应该使用 `re.finditer` 来避免在构建预分词到计数的映射时存储预分词词汇。

**计算 BPE 合并**
将输入文本转换为预分词单元并以 UTF-8 字节表示每个预分词单元后，就可以计算 BPE 合并（即训练 BPE 分词器）了。

BPE 算法的高层逻辑：迭代地统计每个字节对，识别出频率最高的对（"A"，"B"），将所有出现的该最频繁字节对（"A"，"B"）合并，即替换为新 token "AB"。这个新合并的 token 被加入词表；结果，BPE 训练后的最终词表大小等于初始词表大小（我们的情况是256）加上执行的 BPE 合并操作数量。为了训练效率，我们不考虑跨预分词单元边界的字节对。当计算合并时，通过优先选择字典序更大的字节对来确定性地打破平局。例如，如果（"A"，"B"）、（"A"，"C"）、（"B"，"ZZ"）和（"BA"，"A"）都具有最高频率，我们将合并（"BA"，"A"）：

```python
>>> max([("A", "B"), ("A", "C"), ("B", "ZZ"), ("BA", "A")])
('BA', 'A')
```

**特殊 token**
通常，某些字符串（例如 `<|endoftext|>`）用于编码元数据（例如文档之间的边界）。编码文本时，通常希望将某些字符串视为"特殊 token"，它们永远不应被拆分为多个 token（即始终作为单个 token 保留）。例如，序列结束字符串 `<|endoftext|>` 应始终作为单个 token 保留（即单个整数 ID），以便我们知道何时停止从语言模型生成。这些特殊 token 必须加入词表，以便它们有对应的固定 token ID。

**示例（bpe_example）：BPE 训练示例**

以下是 Sennrich et al.（2016）的风格化示例。考虑一个包含以下文本的语料库：

```
low low low low low
lower lower widest widest widest
newest newest newest newest newest newest
```

词表中有一个特殊 token `<|endoftext|>`。

**词表初始化**：我们用特殊 token `<|endoftext|>` 和256个字节值初始化词表。

**预分词**：为简单起见，在本示例中我们假设预分词只是按空格分割。预分词并计数后，得到频率表：

```
{low: 5, lower: 2, widest: 3, newest: 6}
```

方便起见，将其表示为 `dict[tuple[bytes], int]`，例如 `{(l,o,w): 5 …}`。注意，在 Python 中，即使是单个字节也是 `bytes` 对象。Python 中没有 `byte` 类型来表示单个字节，就像 Python 中没有 `char` 类型来表示单个字符一样。

**合并**：我们首先查看每个连续字节对，并对它们出现的词频求和：`{lo: 7, ow: 7, we: 8, er: 2, wi: 3, id: 3, de: 3, es: 9, st: 9, ne: 6, ew: 6}`。字节对 `('es')` 和 `('st')` 并列，所以取字典序更大的对 `('st')`。然后我们会合并预分词单元，得到 `{(l,o,w): 5, (l,o,w,e,r): 2, (w,i,d,e,st): 3, (n,e,w,e,st): 6}`。

第二轮中，`(e, st)` 是最常见的字节对（计数为9），合并后得到 `{(l,o,w): 5, (l,o,w,e,r): 2, (w,i,d,est): 3, (n,e,w,est): 6}`。继续如此，最终得到的合并序列为 `['s t', 'e st', 'o w', 'l ow', 'w est', 'n e', 'ne west', 'w i', 'wi d', 'wid est', 'low e', 'lowe r']`。

取前6次合并，得到 `['s t', 'e st', 'o w', 'l ow', 'w est', 'n e']`，词表元素为 `[<|endoftext|>, [...256字节字符], st, est, ow, low, west, ne]`。

有了这个词表和合并集合，词 `newest` 会被分词为 `[ne, west]`。

**并行化预分词**
你会发现预分词步骤是主要瓶颈。可以使用内置库 `multiprocessing` 并行化代码来加速预分词。具体来说，我们建议在并行实现预分词时，对语料库分块，并确保分块边界出现在特殊 token 的开头。可以直接使用以下链接中的起始代码来获取分块边界：

**预分词前删除特殊 token**
在使用正则表达式模式运行预分词（使用 `re.finditer`）之前，你应该从语料库（或块，如果使用并行实现）中去除所有特殊 token。确保按特殊 token 分割，以便不会跨它们所分隔的文本进行合并。例如，如果语料库（或块）类似于 `[Doc 1]<|endoftext|>[Doc 2]`，你应该按特殊 token `<|endoftext|>` 分割，分别对 `[Doc 1]` 和 `[Doc 2]` 进行预分词，使得不会跨文档边界进行合并。这可以通过使用 `re.split` 以 `"|".join(special_tokens)` 作为分隔符来实现（需要小心使用 `re.escape`，因为 `|` 可能出现在特殊 token 中）。测试 `test_train_bpe_special_tokens` 会测试这一点。

**优化合并步骤**
上述风格化示例中的朴素 BPE 训练实现很慢，因为对于每次合并，它都会遍历所有字节对来识别最频繁的对。然而，每次合并后唯一改变的字节对计数是与合并字节对重叠的那些。因此，通过索引所有字节对的计数并增量更新这些计数，而不是显式遍历每个字节对来统计频率，可以提高 BPE 训练速度。这种缓存过程可以获得显著的加速，但我们注意到 BPE 训练的合并部分在 Python 中无法并行化。

### 2.6 BPE 分词器：编码与解码

在作业的上一部分，我们实现了在输入文本上训练 BPE 分词器的函数，得到分词器词表和 BPE 合并列表。现在，我们将实现一个 BPE 分词器，它加载提供的词表和合并列表，并用它们将文本编码/解码为 token ID。

#### 2.6.1 文本编码

BPE 文本编码的过程与训练 BPE 词表的方式相似，主要有以下几个步骤：

**步骤1：预分词**
首先对序列进行预分词，并将每个预分词单元表示为 UTF-8 字节序列，就像在 BPE 训练中所做的那样。我们将在每个预分词单元内将这些字节合并为词表元素，独立处理每个预分词单元（不跨预分词单元边界合并）。

**步骤2：应用合并**
然后按照 BPE 训练期间创建的词表元素合并序列，按相同的创建顺序应用到预分词单元上。

---

**示例（bpe_encoding）：BPE 编码示例**

假设输入字符串为 `'the cat ate'`，词表为 `{0: b' ', 1: b'a', 2: b'c', 3: b'e', 4: b'h', 5: b't', 6: b'th', 7: b' c', 8: b' a', 9: b'the', 10: b' at'}`，学习到的合并为 `[(b't', b'h'), (b' ', b'c'), (b' ', 'a'), (b'th', b'e'), (b' a', b't')]`。

首先，预分词器将字符串分割为 `['the', ' cat', ' ate']`。然后，我们查看每个预分词单元并应用 BPE 合并。

第一个预分词单元 `'the'` 最初表示为 `[b't', b'h', b'e']`。查看合并列表，识别第一个可应用的合并为 `(b't', b'h')`，将预分词单元转换为 `[b'th', b'e']`。然后回到合并列表，识别下一个可应用的合并为 `(b'th', b'e')`，将预分词单元转换为 `[b'the']`。最后，查看合并列表，发现没有更多适用的合并（因为整个预分词单元已合并为单个 token），完成。对应的整数序列为 `[9]`。

对其余预分词单元重复此过程：预分词单元 `' cat'` 应用 BPE 合并后表示为 `[b' c', b'a', b't']`，整数序列为 `[7, 1, 5]`。最后一个预分词单元 `' ate'` 应用 BPE 合并后为 `[b' at', b'e']`，整数序列为 `[10, 3]`。

因此，编码输入字符串的最终结果为 `[9, 7, 1, 5, 10, 3]`。

---

**特殊 token**：你的分词器应该能够在编码文本时正确处理用户定义的特殊 token（在构造分词器时提供）。

**内存考虑**：假设我们想要对一个无法装入内存的大型文本文件进行分词。为了高效地对这个大文件（或任何其他数据流）进行分词，我们需要将其分成可管理的块并依次处理每个块，使得内存复杂度是常量而不是与文本大小成线性关系。这样做时，需要确保 token 不跨块边界，否则会得到与朴素方法（在内存中对整个序列进行分词）不同的分词结果。

#### 2.6.2 文本解码

要将整数 token ID 序列解码回原始文本，只需查找每个 ID 在词表中对应的条目（字节序列），将它们连接起来，然后将字节解码为 Unicode 字符串即可。注意，输入 ID 不能保证映射到有效的 Unicode 字符串（因为用户可以输入任何整数 ID 序列）。如果输入 token ID 不能产生有效的 Unicode 字符串，应该用官方 Unicode 替换字符 U+FFFD 替换格式错误的字节。`bytes.decode` 的 `errors` 参数控制如何处理 Unicode 解码错误，使用 `errors='replace'` 会自动用替换标记替换格式错误的数据。

## 3 Transformer 语言模型架构

语言模型以批量整数 token ID 序列（即形状为 `(batch_size, sequence_length)` 的 `torch.Tensor`）作为输入，并返回词表上的（批量）归一化概率分布（即形状为 `(batch_size, sequence_length, vocab_size)` 的 PyTorch Tensor），其中预测的分布是每个输入 token 的下一个词。训练语言模型时，我们使用这些下一个词预测来计算实际下一个词与预测的下一个词之间的交叉熵损失。在推理期间从语言模型生成文本时，我们取最后一个时间步的预测下一个词分布（即序列中的最后一项）来生成序列中的下一个 token（例如，取概率最高的 token、从分布中采样等），将生成的 token 添加到输入序列中，然后重复。

在作业的这一部分，你将从头构建这个 Transformer 语言模型。我们将从模型的高层描述开始，逐步详细介绍各个组件。

### 3.1 Transformer LM

给定一个 token ID 序列，Transformer 语言模型使用输入嵌入将 token ID 转换为稠密向量，将嵌入的 token 通过 `num_layers` 个 Transformer 块，然后应用一个学习到的线性投影（"输出嵌入"或"LM 头"）来产生预测的下一个 token logits。

#### 3.1.1 Token 嵌入

第一步，Transformer 将（批量）token ID 序列嵌入为包含 token 身份信息的向量序列（图1中的红色块）。

具体来说，给定一个 token ID 序列，Transformer 语言模型使用 token 嵌入层来产生向量序列。每个嵌入层接受形状为 `(batch_size, sequence_length)` 的整数张量，并产生形状为 `(batch_size, sequence_length, d_model)` 的向量序列。

#### 3.1.2 Pre-norm Transformer 块

嵌入后，激活值被若干个结构相同的神经网络层处理。标准的仅解码器 Transformer 语言模型由 `num_layers` 个相同的层（通常称为 Transformer "块"）组成。每个 Transformer 块接受形状为 `(batch_size, sequence_length, d_model)` 的输入，并返回形状为 `(batch_size, sequence_length, d_model)` 的输出。每个块通过自注意力跨序列聚合信息，并通过前馈层对其进行非线性变换。

### 3.2 输出归一化和嵌入

经过 `num_layers` 个 Transformer 块后，我们将最终激活值转换为词表上的分布。

我们将实现"pre-norm" Transformer 块（详见 §3.5），这还需要在最后一个 Transformer 块之后使用层归一化，以确保其输出被适当缩放。

归一化之后，我们将使用标准的学习线性变换将 Transformer 块的输出转换为预测的下一个 token logits。

### 3.3 关于批处理、Einsum 和高效计算的备注

在整个 Transformer 中，我们将对许多类似批次的输入执行相同的计算。以下是几个示例：

- **批次元素**：我们对每个批次元素应用相同的 Transformer 前向操作。
- **序列长度**："逐位置"操作（如 RMSNorm 和前馈）在序列的每个位置上独立运行。
- **注意力头**：注意力操作在多头"multi-headed"注意力操作中跨注意力头批量化。

拥有一种符合人体工学的方式来执行这些操作是很有用的，这种方式可以充分利用 GPU，并且易于阅读和理解。许多 PyTorch 操作可以在张量开头接受额外的"类批次"维度，并在这些维度上高效地重复/广播操作。

我们强烈建议在课程中学习和使用 einsum 符号（`torch.einsum`）或框架无关库（如 `einops` 或 `einx`）。两个关键操作是 `einsum`（可以对任意维度的输入张量进行张量收缩）和 `rearrange`（可以重新排列、连接和分割任意维度）。

**示例（einstein_example1）：使用 einops.einsum 进行批量矩阵乘法**

```python
import torch
from einops import rearrange, einsum
## 基本实现
Y = D @ A.T
# 难以判断输入和输出形状以及它们的含义。

## Einsum 是自文档化的且健壮的
Y = einsum(D, A, "batch sequence d_in, d_out d_in -> batch sequence d_out")
## 或者，D 可以有任意前导维度但 A 受约束的批量版本。
Y = einsum(D, A, "... d_in, d_out d_in -> ... d_out")
```

**示例（einstein_example2）：使用 einops.rearrange 进行广播操作**

```python
images = torch.randn(64, 128, 128, 3) # (batch, height, width, channel)
dim_by = torch.linspace(start=0.0, end=1.0, steps=10)
## 重塑并相乘
dim_value = rearrange(dim_by, "dim_value -> 1 dim_value 1 1 1")
images_rearr = rearrange(images, "b height width channel -> b 1 height width channel")
dimmed_images = images_rearr * dim_value
## 或者一步完成：
dimmed_images = einsum(
    images, dim_by,
    "batch height width channel, dim_value -> batch dim_value height width channel"
)
```

**示例（einstein_example3）：使用 einops.rearrange 进行像素混合**

```python
channels_last = torch.randn(64, 32, 32, 3)  # (batch, height, width, channel)
B = torch.randn(32*32, 32*32)

# 使用 einops 的版本
height = width = 32
channels_first = rearrange(
    channels_last,
    "batch height width channel -> batch channel (height width)"
)
channels_first_transformed = einsum(
    channels_first, B,
    "batch channel pixel_in, pixel_out pixel_in -> batch channel pixel_out"
)
channels_last_transformed = rearrange(
    channels_first_transformed,
    "batch channel (height width) -> batch height width channel",
    height=height, width=width
)
```

#### 3.3.1 数学符号和内存顺序

许多机器学习论文在符号中使用行向量，这与 NumPy 和 PyTorch 默认使用的行优先（row-major）内存顺序配合良好。对于行向量，线性变换表示为：

$$y = xW^\top$$

在本作业中，我们将使用列向量进行数学符号表示，因为这样通常更容易理解数学。请记住，如果你想使用普通矩阵乘法符号，你需要使用行向量约定来应用矩阵，因为 PyTorch 使用行优先内存顺序。如果你将 einsum 用于矩阵运算，这不是问题。

### 3.5 Pre-Norm Transformer 块

每个 Transformer 块有两个子层：多头自注意力机制和逐位置前馈网络（Vaswani et al., 2017，第3.1节）。

在原始 Transformer 论文中，模型在每个两个子层周围使用残差连接，然后进行层归一化。这种架构通常称为"post-norm" Transformer，因为层归一化应用于子层输出。然而，一系列工作发现，将层归一化从每个子层的输出移到每个子层的输入（在最后一个 Transformer 块之后再额外添加一个层归一化）可以改善 Transformer 训练稳定性——这就是"pre-norm" Transformer 块（见图2）。每个 Transformer 块子层的输出通过残差连接加到子层输入上。pre-norm 的一个直觉是，有一条从输入嵌入到 Transformer 最终输出的清晰"残差流"，没有任何归一化，据称可以改善梯度流动。这种 pre-norm Transformer 现在是当今语言模型（例如 GPT-3、LLaMA、PaLM 等）使用的标准。我们将实现这种变体。

#### 3.5.1 均方根层归一化（RMSNorm）

原始 Vaswani et al.（2017）的 Transformer 实现使用层归一化来归一化激活值。遵循 Touvron et al.（2023），我们将使用均方根层归一化（RMSNorm；Zhang and Sennrich, 2019，公式4）进行层归一化。给定激活值向量 $a \in \mathbb{R}^{d_{model}}$，RMSNorm 将每个激活值 $a_i$ 重新缩放为：

$$\text{RMSNorm}(a_i) = \frac{a_i}{\text{RMS}(a)} g_i$$

其中 $\text{RMS}(a) = \sqrt{\frac{1}{d_{model}} \sum_{i=1}^{d_{model}} a_i^2 + \varepsilon}$。这里，$g_i$ 是可学习的"增益"参数（共有 `d_model` 个这样的参数），$\varepsilon$ 是通常固定为 `1e-5` 的超参数。

你应该将输入上溯（upcast）为 `torch.float32` 以防止对输入进行平方时溢出。整体上，你的 `forward` 方法应该如下所示：

```python
in_dtype = x.dtype
x = x.to(torch.float32)
# 在这里执行 RMSNorm 的代码
...
result = ...
# 以原始 dtype 返回结果
return result.to(in_dtype)
```

- `d_model: int` — 模型的隐藏维度
- `eps: float = 1e-5` — 数值稳定性的 epsilon 值
- `device: torch.device | None = None`


#### 3.5.2 逐位置前馈网络（SwiGLU）

在原始 Transformer 论文（Vaswani et al., 2017，第3.3节）中，Transformer 前馈网络由两个线性变换组成，中间有一个 ReLU 激活函数。内层前馈层的维度通常是输入维度的4倍。

然而，现代语言模型倾向于与这种原始设计有两个主要变化：它们使用另一种激活函数并采用门控机制。具体来说，我们将实现"SwiGLU"激活函数，该函数将 SiLU（通常称为 Swish）激活与称为门控线性单元（GLU）的门控机制结合。

SiLU 或 Swish 激活函数定义为：

$$\text{SiLU}(x) = x \cdot \sigma(x) = \frac{x}{1 + e^{-x}}$$

门控线性单元（GLU）定义为：

$$\text{GLU}(x, W_1, W_2) = \sigma(W_1 x) \odot W_2 x$$

将 SiLU/Swish 和 GLU 结合，得到 SwiGLU：

$$\text{FFN}(x) = \text{SwiGLU}(x, W_1, W_2, W_3) = W_2(\text{SiLU}(W_1 x) \odot W_3 x)$$

其中 $x \in \mathbb{R}^{d_{model}}$，$W_1, W_3 \in \mathbb{R}^{d_{ff} \times d_{model}}$，$W_2 \in \mathbb{R}^{d_{model} \times d_{ff}}$，且通常 $d_{ff} = \frac{8}{3} d_{model}$。

**问题（positionwise_feedforward）：实现逐位置前馈网络（2分）**

**交付物**：实现由 SiLU 激活函数和 GLU 组成的 SwiGLU 前馈网络。

注意：在此特定情况下，可以在实现中使用 `torch.sigmoid` 以确保数值稳定性。

你应该将 $d_{ff}$ 设置为约 $\frac{8}{3} \times d_{model}$，同时确保内层前馈层的维度是64的倍数，以充分利用硬件。要对照提供的测试测试你的实现，需要实现 `[adapters.run_swiglu]` 中的测试适配器，然后运行 `uv run pytest -k test_swiglu`。

#### 3.5.3 相对位置嵌入（RoPE）

为了向模型注入位置信息，我们将实现旋转位置嵌入（Rotary Position Embeddings；Su et al., 2021），通常称为 RoPE。

对于给定的查询 token $q^{(i)} = W_q x^{(i)} \in \mathbb{R}^d$（位于位置 $i$），我们将应用一个逐对旋转矩阵 $R_i$，得到 $q'^{(i)} = R_i q^{(i)} = R_i W_q x^{(i)}$。

这里，$R_i$ 将嵌入元素对 $q^{(i)}_{2k-1:2k}$ 旋转 $\theta_{i,k} = \frac{i}{\Theta^{(2k-2)/d}}$ 角度（对 $k \in \{1, \ldots, d/2\}$ 和某个常数 $\Theta$）。因此，$R_i$ 可以被视为大小为 $d \times d$ 的块对角矩阵，其中块 $R_k^i$ 为：

$$R_k^i = \begin{bmatrix} \cos(\theta_{i,k}) & -\sin(\theta_{i,k}) \\ \sin(\theta_{i,k}) & \cos(\theta_{i,k}) \end{bmatrix}$$

这一层没有可学习的参数。

#### 3.5.4 缩放点积注意力（Scaled Dot-Product Attention）

我们将按照 Vaswani et al.（2017）第3.2.1节实现缩放点积注意力。

首先，注意力操作的定义将使用 softmax，该操作将未归一化的分数向量转换为归一化分布：

$$\text{softmax}(v)_i = \frac{\exp(v_i)}{\sum_{j=1}^n \exp(v_j)}$$

注意，对于较大的值，$\exp(v_i)$ 可能变为 `inf`（然后 `inf/inf = NaN`）。我们可以通过注意到 softmax 操作对所有输入加上任何常数 $c$ 不变来避免这个问题。通常，我们会从所有元素中减去 $o_i$ 的最大条目，使新的最大条目为0。

注意力操作定义为：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right) V$$

其中 $Q \in \mathbb{R}^{n \times d_k}$，$K \in \mathbb{R}^{m \times d_k}$，$V \in \mathbb{R}^{m \times d_v}$。

**掩码（Masking）**：有时方便地对注意力操作的输出进行掩码。掩码的形状为 $M \in \{\text{True}, \text{False}\}^{n \times m}$，每行 $i$ 表示查询 $i$ 应关注哪些键。`True` 值表示查询 $i$ 关注键 $j$，`False` 表示不关注。计算上，通过在掩码矩阵中 `False` 的任何条目处添加 $-\infty$，比对子序列计算注意力效率高得多。

**问题（softmax）：实现 softmax（1分）**

**交付物**：编写一个函数来对张量应用 softmax 操作。你的函数应该接受两个参数：一个张量和一个维度 $i$，并对输入张量的第 $i$ 个维度应用 softmax。使用从第 $i$ 个维度的所有元素中减去最大值的技巧来避免数值稳定性问题。

#### 3.5.5 因果多头自注意力（Causal Multi-Head Self-Attention）

我们将按照 Vaswani et al.（2017）第3.2.2节实现多头自注意力。多头注意力操作的定义为：

$$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h)$$
$$\text{head}_i = \text{Attention}(Q_i, K_i, V_i)$$

多头自注意力操作：

$$\text{MultiHeadSelfAttention}(x) = W^O \text{MultiHead}(W^Q x, W^K x, W^V x)$$

其中可学习参数 $W^Q \in \mathbb{R}^{hd_k \times d_{model}}$，$W^K \in \mathbb{R}^{hd_k \times d_{model}}$，$W^V \in \mathbb{R}^{hd_v \times d_{model}}$，$W^O \in \mathbb{R}^{d_{model} \times hd_v}$。

**因果掩码**：你的实现应防止模型关注序列中的未来 token。对于输入 token 序列 $t_1, \ldots, t_n$，我们可以使用因果注意力掩码，允许 token $i$ 关注序列中所有位置 $j \leq i$。可以使用 `torch.triu` 或广播索引比较来构造此掩码。

**应用 RoPE**：RoPE 应应用于查询和键向量，但不应用于值向量。此外，头部维度应作为批次维度处理，因为在多头注意力中，注意力对每个头独立应用。

**问题（multihead_self_attention）：实现因果多头自注意力（5分）**

### 4.1 交叉熵损失

Transformer 语言模型定义了分布 $p_\theta(x_{i+1} | x_{1:i})$，对于长度为 $m+1$ 的每个序列 $x$ 和 $i = 1, \ldots, m$。给定由长度为 $m$ 的序列组成的训练集 $\mathcal{D}$，我们定义标准交叉熵（负对数似然）损失函数：

$$\ell(\theta; \mathcal{D}) = \frac{1}{|\mathcal{D}|m} \sum_{x \in \mathcal{D}} \sum_{i=1}^m - \log p_\theta(x_{i+1} | x_{1:i})$$

Transformer 为每个位置 $i$ 计算 logits $o_i \in \mathbb{R}^{vocab\_size}$，从而得到：

$$p(x_{i+1} | x_{1:i}) = \text{softmax}(o_i)[x_{i+1}] = \frac{\exp(o_i[x_{i+1}])}{\sum_{a=1}^{vocab\_size} \exp(o_i[a])}$$

**问题（cross_entropy）：实现交叉熵（未注分数）**

- 减去最大元素以确保数值稳定性。
- 尽可能抵消 log 和 exp。
- 处理任何额外的批次维度，并返回批次的平均值。

$$\text{perplexity} = \exp\left(\frac{1}{m} \sum_{i=1}^m \ell_i\right)$$

### 4.2 SGD 优化器

有了损失函数，我们开始探索优化器。最简单的基于梯度的优化器是随机梯度下降（SGD）。我们从随机初始化的参数 $\theta_0$ 开始，对每一步 $t = 0, \ldots, T-1$ 执行以下更新：

$$\theta_{t+1} \leftarrow \theta_t - \alpha_t \nabla L(\theta_t; B_t)$$

其中 $B_t$ 是从数据集 $\mathcal{D}$ 中随机采样的批次，学习率 $\alpha_t$ 和批次大小 $|B_t|$ 是超参数。

#### 4.2.1 在 PyTorch 中实现 SGD

要实现我们的优化器，我们将继承 PyTorch 的 `torch.optim.Optimizer` 类。`Optimizer` 子类必须实现两个方法：

- `def __init__(self, params, ...)` — 初始化优化器。
- `def step(self)` — 执行参数的一次更新。

以下是带有学习率衰减的 SGD 变体（作为示例）：

$$\theta_{t+1} = \theta_t - \frac{\alpha}{\sqrt{t+1}} \nabla L(\theta_t; B_t)$$

```python
from collections.abc import Callable, Iterable
from typing import Optional
import torch
import math

class SGD(torch.optim.Optimizer):
    def __init__(self, params, lr=1e-3):
        if lr < 0:
            raise ValueError(f"Invalid learning rate: {lr}")
        defaults = {"lr": lr}
        super().__init__(params, defaults)

    def step(self, closure: Optional[Callable] = None):
        loss = None if closure is None else closure()
        for group in self.param_groups:
            lr = group["lr"]  # 获取学习率
            for p in group["params"]:
                if p.grad is None:
                    continue
                state = self.state[p]  # 获取与 p 关联的状态
                t = state.get("t", 0)  # 从状态获取迭代次数，或初始值
                grad = p.grad.data  # 获取 p 对应的损失梯度
                p.data -= lr / math.sqrt(t + 1) * grad  # 原地更新权重张量
                state["t"] = t + 1  # 增加迭代次数
        return loss
```

### 4.3 AdamW

现代语言模型通常用更复杂的优化器训练，而不是 SGD。最近使用的大多数优化器是 Adam 优化器（Kingma and Ba, 2015）的衍生品。我们将使用 AdamW（Loshchilov and Hutter, 2019），它在最近的工作中被广泛使用。AdamW 提出了对 Adam 的修改，通过添加权重衰减（在每次迭代中，我们将参数拉向0）来改善正则化，这种方式与梯度更新解耦。

AdamW 是有状态的：对于每个参数，它跟踪其第一和第二矩的运行估计。因此，AdamW 以额外内存换取更好的稳定性和收敛性。

AdamW 算法（其中 $\varepsilon$ 是一个小值，例如 $10^{-8}$）：

```
初始化(θ)
m ← 0  （第一矩向量初始值）
v ← 0  （第二矩向量初始值）

对于 t = 1, ..., T:
  采样批次 Bt
  g ← ∇θ ℓ(θ; Bt)          （计算当前步的梯度）
  m ← β1·m + (1-β1)·g       （更新第一矩估计）
  v ← β2·v + (1-β2)·g²      （更新第二矩估计）
  αt ← α · √(1-(β2)^t) / (1-(β1)^t)   （计算调整后的 α）
  θ ← θ - αt · m / (√v + ε) （更新参数）
  θ ← θ - α·λ·θ              （应用权重衰减）
```

注意 $t$ 从1开始。典型的超参数设置：$(\beta_1, \beta_2) = (0.9, 0.999)$，但 LLaMA 和 GPT-3 等大型语言模型通常使用 $(0.9, 0.95)$。

### 4.4 学习率调度

在训练 Transformer 时，通常使用学习率调度：开始时使用较大的学习率进行更快的更新，随着模型训练逐渐衰减到较小的值。在本作业中，我们将实现用于训练 LLaMA（Touvron et al., 2023）的余弦退火调度（cosine annealing schedule）。

余弦退火学习率调度接受：(i) 当前迭代 $t$，(ii) 最大学习率 $\alpha_{max}$，(iii) 最小（最终）学习率 $\alpha_{min}$，(iv) 预热迭代次数 $T_w$，以及 (v) 余弦退火迭代次数 $T_c$。迭代 $t$ 的学习率定义为：

- **预热（Warm-up）**：如果 $t < T_w$，则 $\alpha_t = \frac{t}{T_w} \alpha_{max}$
- **余弦退火**：如果 $T_w \leq t \leq T_c$，则 $\alpha_t = \alpha_{min} + \frac{1}{2}\left(1 + \cos\left(\frac{\pi(t-T_w)}{T_c-T_w}\right)\right)(\alpha_{max} - \alpha_{min})$
- **退火后**：如果 $t > T_c$，则 $\alpha_t = \alpha_{min}$

**问题（learning_rate_schedule）：实现带预热的余弦学习率调度（未注分数）**

编写一个函数，接受 $t$、$\alpha_{max}$、$\alpha_{min}$、$T_w$ 和 $T_c$，并根据上述调度器定义返回学习率 $\alpha_t$。然后实现 `[adapters.get_lr_cosine_schedule]` 并确保它通过 `uv run pytest -k test_get_lr_cosine_schedule`。

### 4.5 梯度裁剪（Gradient Clipping）

在训练过程中，有时会遇到产生大梯度的训练样本，这可能使训练不稳定。为了缓解这个问题，通常在实践中使用梯度裁剪。其思想是在每次反向传播后、执行优化器步骤之前，对梯度的范数施加限制。

给定所有参数的梯度 $g$，我们计算其 $\ell_2$-范数 $\|g\|_2$。如果此范数小于最大值 $M$，则保持 $g$ 不变；否则，将 $g$ 按因子 $\frac{M}{\|g\|_2 + \varepsilon}$ 缩放（其中添加一个小的 $\varepsilon$，如 $10^{-6}$，以确保数值稳定性）。

**问题（gradient_clipping）：实现梯度裁剪（1分）**

编写一个实现梯度裁剪的函数。你的函数应接受参数列表和最大 $\ell_2$-范数，并原地修改每个参数的梯度。使用 $\varepsilon = 10^{-6}$（PyTorch 默认值）。然后实现适配器 `[adapters.run_gradient_clipping]` 并确保它通过 `uv run pytest -k test_gradient_clipping`。

### 5.2 检查点（Checkpointing）

## 6 生成文本

现在我们可以训练模型了，我们需要的最后一块是从模型生成文本的能力。

回顾一下，语言模型接受（可能批量的）长度为 `sequence_length` 的整数序列，并产生大小为 `(sequence_length × vocab_size)` 的矩阵，其中序列的每个元素都是预测该位置之后下一个词的概率分布。我们现在将编写几个函数，将其转换为新序列的采样方案。

**Softmax**：按照标准约定，语言模型输出是最后一个线性层的输出（"logits"），因此我们必须通过 softmax 操作将其转换为归一化概率。

**解码（Decoding）**：要从模型生成文本（解码），我们将向模型提供前缀 token 序列（"提示"），并要求它产生预测序列中下一个词的词表概率分布。然后，我们将从这个分布中采样来确定下一个输出 token。

解码过程的一步应接受序列 $x_{1\ldots t}$ 并通过以下公式返回 token $x_{t+1}$：

$$P(x_{t+1} = i | x_{1\ldots t}) = \frac{\exp(v_i)}{\sum_j \exp(v_j)}$$
$$v = \text{TransformerLM}(x_{1\ldots t})_t \in \mathbb{R}^{vocab\_size}$$

**解码技巧**：

1. **温度缩放（Temperature Scaling）**：我们用温度参数 $\tau$ 修改 softmax：

$$\text{softmax}(v, \tau)_i = \frac{\exp(v_i/\tau)}{\sum_{j=1}^{|vocab\_size|} \exp(v_j/\tau)}$$

当 $\tau \to 0$ 时，$v$ 中的最大元素主导，softmax 输出变为集中在最大元素的 one-hot 向量。

2. **核采样（Nucleus/Top-p Sampling）**：通过截断低概率词来修改采样分布。设 $q$ 为从（温度缩放的）softmax 得到的概率分布（大小为 `vocab_size`）。使用超参数 $p$ 的核采样根据以下公式产生下一个 token：

$$P(x_{t+1} = i | q) = \begin{cases} \frac{q_i}{\sum_{j \in V(p)} q_j} & \text{如果 } i \in V(p) \\ 0 & \text{其他} \end{cases}$$

其中 $V(p)$ 是满足 $\sum_{j \in V(p)} q_j \geq p$ 的最小索引集合。

**问题（decoding）：解码（3分）**

**交付物**：实现一个从语言模型解码的函数。建议支持以下功能：

- 为用户提供的提示生成补全（即接受一些 $x_{1\ldots t}$ 并采样补全，直到遇到 `<|endoftext|>` token）。
- 允许用户控制生成的最大 token 数量。
- 给定所需的温度值，在采样之前对预测的下一个词分布应用 softmax 温度缩放。
- Top-p 采样（Holtzman et al., 2020；也称为核采样），给定用户指定的阈值值。

## 3 Transformer 语言模型架构

语言模型（language model，LM）以批量整数词元 ID 序列（即形状为 `(batch_size, sequence_length)` 的 `torch.Tensor`）作为输入，并返回词汇表上的（批量）归一化概率分布（即形状为 `(batch_size, sequence_length, vocab_size)` 的 PyTorch 张量），其中预测分布是对每个输入词元的下一个词的预测。在训练语言模型时，我们使用这些下一词预测来计算实际下一词与预测下一词之间的交叉熵损失。在推理期间从语言模型生成文本时，我们取最后一个时间步（即序列中的最后一项）的预测下一词分布，从中生成下一个词元（例如，通过取概率最高的词元，或从分布中采样等），将生成的词元添加到输入序列中，然后重复。

在这部分作业中，你将从头开始构建这个 Transformer 语言模型。我们将从模型的高层描述开始，然后逐步详细说明各个组件。

### 3.1 Transformer LM

给定词元 ID 序列，Transformer 语言模型使用输入嵌入将词元 ID 转换为密集向量，将嵌入的词元通过 `num_layers` 个 Transformer 块，然后应用学习的线性投影（"输出嵌入"或"LM 头"）来生成预测的下一词元 logits。参见图 1 的示意图。

#### 3.1.1 词元嵌入（Token Embeddings）

在第一步，Transformer 将（批量的）词元 ID 序列嵌入到包含词元身份信息的向量序列中（图 1 中的红色块）。

更具体地说，给定词元 ID 序列，Transformer 语言模型使用词元嵌入层产生向量序列。每个嵌入层接受形状为 `(batch_size, sequence_length)` 的整数张量，并产生形状为 `(batch_size, sequence_length, d_model)` 的向量序列。

#### 3.1.2 预归一化 Transformer 块（Pre-norm Transformer Block）

嵌入后，激活值由几个结构相同的神经网络层处理。标准的仅解码器 Transformer 语言模型由 `num_layers` 个相同的层（通常称为 Transformer "块"）组成。每个 Transformer 块接受形状为 `(batch_size, sequence_length, d_model)` 的输入，返回形状为 `(batch_size, sequence_length, d_model)` 的输出。每个块通过自注意力聚合序列中的信息，并通过前馈层进行非线性变换。

### 3.2 输出归一化和嵌入（Output Normalization and Embedding）

经过 `num_layers` 个 Transformer 块后，我们将取最终激活值并将其转换为词汇表上的分布。我们将实现"预归一化"Transformer 块（详见 §3.5），这还需要在最后一个 Transformer 块之后使用层归一化，以确保其输出被适当缩放。在此归一化之后，我们将使用标准的学习线性变换，将 Transformer 块的输出转换为预测的下一词元 logits（参见例如 Radford 等人 [2018] 公式 2）。

### 3.3 备注：批处理、Einsum 和高效计算（Remark: Batching, Einsum and Efficient Computation）

在整个 Transformer 中，我们将对许多类批量输入执行相同的计算。以下是几个例子：
- 批次元素：我们对每个批次元素应用相同的 Transformer 前向操作。
- 序列长度："逐位置"操作如 RMSNorm 和前馈对序列的每个位置独立执行相同操作。
- 注意力头：注意力操作在多头注意力操作中跨注意力头批量执行。

以一种充分利用 GPU 且易于阅读和理解的方式执行此类操作是很有用的。许多 PyTorch 操作可以在张量开头接受额外的"类批量"维度，并在这些维度上高效地重复/广播操作。

例如，假设我们正在执行一个逐位置的批量操作。我们有一个形状为 `(batch_size, sequence_length, d_model)` 的"数据张量" D，想要对形状为 `(d_model, d_model)` 的矩阵 A 进行批量向量矩阵乘法。在这种情况下，`D @ A` 将执行批量矩阵乘法，这是 PyTorch 中的高效原语，其中 `(batch_size, sequence_length)` 维度被批量处理。

因此，最好假设你的函数可能会收到额外的类批量维度，并将这些维度保留在 PyTorch 形状的开头。为了组织张量以便以这种方式批量处理，可能需要通过多步 `view`、`reshape` 和 `transpose` 来调整形状。这可能有些麻烦，而且通常很难读懂代码在做什么以及张量的形状是什么。

更符合人机工效学的选择是在 `torch.einsum` 中使用 einsum 符号，或者使用框架无关的库如 `einops` 或 `einx`。两个关键操作是 `einsum`（可以对任意维度的输入张量进行张量收缩）和 `rearrange`（可以重新排序、连接和拆分任意维度）。事实证明，机器学习中几乎所有操作都是维度调整和张量收缩的某种组合，偶尔使用（通常是逐点的）非线性函数。这意味着，当使用 einsum 符号时，很多代码会更具可读性和灵活性。

**我们强烈建议在课程中学习并使用 einsum 符号。** 以前未接触过 einsum 符号的学生应使用 `einops`，已熟悉 `einops` 的学生应学习更通用的 `einx`。两个包在我们提供的环境中都已安装。

以下是一些如何使用 einsum 符号的示例。

**示例 (einstein_example1): 使用 einops.einsum 进行批量矩阵乘法**

```python
import torch
from einops import rearrange, einsum
## 基本实现
Y = D @ A.T
## Einsum 是自文档化且健壮的
Y = einsum(D, A, "batch sequence d_in, d_out d_in -> batch sequence d_out")
## 或者，一个批量化版本，其中 D 可以有任意前导维度，但 A 受到约束。
Y = einsum(D, A, "... d_in, d_out d_in -> ... d_out")
```

**示例 (einstein_example2): 使用 einops.rearrange 进行广播操作**

```python
images = torch.randn(64, 128, 128, 3) # (batch, height, width, channel)
dim_by = torch.linspace(start=0.0, end=1.0, steps=10)
dim_value = rearrange(dim_by, "dim_value -> 1 dim_value 1 1 1")
images_rearr = rearrange(images, "b height width channel -> b 1 height width channel")
dimmed_images = images_rearr * dim_value
## 或者一步完成：
dimmed_images = einsum(
    images, dim_by,
    "batch height width channel, dim_value -> batch dim_value height width channel"
)
```

**示例 (einstein_example3): 使用 einops.rearrange 进行像素混合**

```python
channels_last = torch.randn(64, 32, 32, 3)
B = torch.randn(32*32, 32*32)
# 使用 einops：
height = width = 32
channels_first = rearrange(
    channels_last,
    "batch height width channel -> batch channel (height width)"
)
channels_first_transformed = einsum(
    channels_first, B,
    "batch channel pixel_in, pixel_out pixel_in -> batch channel pixel_out"
)
channels_last_transformed = rearrange(
    channels_first_transformed,
    "batch channel (height width) -> batch height width channel",
    height=height, width=width
)
```

使用 einsum 符号时，文档即实现！Einsum 符号可以处理任意输入批处理维度，并且有自文档化的关键好处。

#### 3.3.1 数学符号和内存排序（Mathematical Notation and Memory Ordering）

许多机器学习论文在其符号中使用行向量，这与 NumPy 和 PyTorch 默认使用的行优先内存排序配合良好。使用行向量时，线性变换看起来像：

$$y = xW^\top,$$

对于行优先的 $W \in \mathbb{R}^{d_{out} \times d_{in}}$ 和行向量 $x \in \mathbb{R}^{1 \times d_{in}}$。

在线性代数中，通常更常见使用列向量，线性变换看起来像：

$$y = Wx,$$

本作业将使用列向量进行数学符号。你应该记住，如果想使用普通矩阵乘法符号，需要使用行向量约定来应用矩阵，因为 PyTorch 使用行优先内存排序。如果对矩阵操作使用 einsum，这应该不是问题。

### 3.5 预归一化 Transformer 块（Pre-Norm Transformer Block）

每个 Transformer 块有两个子层：多头自注意力机制和逐位置前馈网络（Vaswani 等人，2017，第 3.1 节）。

在原始 Transformer 论文中，模型在每两个子层周围使用残差连接，然后是层归一化。这种架构通常被称为"后归一化"Transformer（post-norm Transformer）。然而，多项研究发现，将层归一化从每个子层的输出移动到每个子层的输入可以提高 Transformer 训练稳定性 [Nguyen and Salazar, 2019, Xiong et al., 2020]——参见图 2 的"预归一化"Transformer 块的可视化表示。每个 Transformer 块子层的输出通过残差连接添加到子层输入（Vaswani 等人，2017，第 5.4 节）。预归一化的直觉在于，从输入嵌入到 Transformer 最终输出之间有一个干净的"残差流"，没有任何归一化，据说这可以改善梯度流。这种预归一化 Transformer 现在是当今语言模型中使用的标准（例如 GPT-3、LLaMA、PaLM 等），因此我们将实现这个变体。我们将逐步完成预归一化 Transformer 块的每个组件并按顺序实现它们。

#### 3.5.1 均方根层归一化（Root Mean Square Layer Normalization）

Vaswani 等人 [2017] 的原始 Transformer 实现使用层归一化（layer normalization）[Ba et al., 2016] 来归一化激活值。遵循 Touvron 等人 [2023]，我们将使用均方根层归一化（RMSNorm；Zhang and Sennrich, 2019，公式 4）进行层归一化。给定激活值向量 $a \in \mathbb{R}^{d_{model}}$，RMSNorm 将每个激活值 $a_i$ 重缩放如下：

$$\text{RMSNorm}(a_i) = \frac{a_i}{\text{RMS}(a)} g_i,$$

其中 $\text{RMS}(a) = \sqrt{\frac{1}{d_{model}} \sum_{i=1}^{d_{model}} a_i^2 + \varepsilon}$。这里，$g_i$ 是可学习的"增益"参数（共有 `d_model` 个这样的参数），$\varepsilon$ 是一个通常固定为 `1e-5` 的超参数。

应将输入上转型为 `torch.float32` 以防止在对输入求平方时溢出。总体上，`forward` 方法应如下所示：

```python
in_dtype = x.dtype
x = x.to(torch.float32)
# 执行 RMSNorm 的代码
...
result = ...
# 以原始数据类型返回结果
return result.to(in_dtype)
```

- `d_model: int` 模型的隐藏维度
- `eps: float = 1e-5` 数值稳定性的 epsilon 值
- `device`, `dtype`: 可选

`def forward(self, x: torch.Tensor) -> torch.Tensor` 处理形状为 `(batch_size, sequence_length, d_model)` 的输入张量，返回相同形状的张量。

注意：在执行归一化之前将输入上转型为 `torch.float32`（之后再下转型为原始数据类型），如上所述。

#### 3.5.2 逐位置前馈网络（Position-Wise Feed-Forward Network）

在原始 Transformer 论文（Vaswani 等人 [2017] 第 3.3 节）中，Transformer 前馈网络由两个线性变换组成，中间有一个 ReLU 激活（$\text{ReLU}(x) = \max(0, x)$）。内部前馈层的维度通常是输入维度的 4 倍。

然而，现代语言模型与这种原始设计相比有两个主要变化：它们使用另一种激活函数并采用门控机制。具体来说，我们将实现"SwiGLU"激活函数，该函数被 Llama 3 [Grattafiori 等人，2024] 和 Qwen 2.5 [Yang 等人，2024] 等 LLM 所采用，它结合了 SiLU（通常称为 Swish）激活与称为门控线性单元（Gated Linear Unit，GLU）的门控机制。我们还将省略线性层中有时使用的偏置项，遵循自 PaLM [Chowdhery 等人，2022] 和 LLaMA [Touvron 等人，2023] 以来的大多数现代 LLM。

SiLU（Sigmoid-Weighted Linear Unit）或 Swish 激活函数 [Hendrycks and Gimpel, 2016, Elfwing 等人，2017] 定义如下：

$$\text{SiLU}(x) = x \cdot \sigma(x) = \frac{x}{1 + e^{-x}}$$

如图 3 所示，SiLU 激活函数类似于 ReLU 激活函数，但在零处是光滑的。

门控线性单元（GLU）最初由 Dauphin 等人 [2017] 定义为通过 sigmoid 函数的线性变换与另一个线性变换的逐元素乘积：

$$\text{GLU}(x, W_1, W_2) = \sigma(W_1 x) \odot W_2 x,$$

其中 $\odot$ 表示逐元素乘法。门控线性单元被建议"通过为梯度提供线性路径同时保留非线性能力，来减少深层架构的梯度消失问题"。

将 SiLU/Swish 和 GLU 放在一起，我们得到 SwiGLU，将用于前馈网络：

$$\text{FFN}(x) = \text{SwiGLU}(x, W_1, W_2, W_3) = W_2(\text{SiLU}(W_1 x) \odot W_3 x),$$

其中 $x \in \mathbb{R}^{d_{model}}$，$W_1, W_3 \in \mathbb{R}^{d_{ff} \times d_{model}}$，$W_2 \in \mathbb{R}^{d_{model} \times d_{ff}}$，正则情况下 $d_{ff} = \frac{8}{3} d_{model}$。

Shazeer [2020] 首先提出将 SiLU/Swish 激活与 GLU 结合，并进行实验表明 SwiGLU 在语言建模任务上优于 ReLU 和 SiLU（无门控）等基线。稍后在作业中，你将比较 SwiGLU 和 SiLU。Shazeer 论文中一句现在著名的话是：

> 我们无法解释为什么这些架构似乎有效；我们将其成功，如一切其他事情，归因于神圣的仁慈。

**Problem (positionwise_feedforward): 实现逐位置前馈网络** (2 points)

**可交付成果：** 实现 SwiGLU 前馈网络，由 SiLU 激活函数和 GLU 组成。

注意：为了数值稳定性，在实现中可以自由使用 `torch.sigmoid`。

在实现中，应将 $d_{ff}$ 设置为约 $\frac{8}{3} \times d_{model}$，同时确保内部前馈层的维度是 64 的倍数以充分利用硬件。要针对我们提供的测试来测试你的实现，需要在 `[adapters.run_swiglu]` 处实现测试适配器。然后，运行 `uv run pytest -k test_swiglu` 来测试你的实现。

#### 3.5.3 相对位置嵌入（Relative Positional Embeddings）

为了将位置信息注入模型，我们将实现旋转位置嵌入（Rotary Position Embeddings，RoPE）[Su 等人，2021]。对于给定查询词元 $q^{(i)} = W_q x^{(i)} \in \mathbb{R}^d$ 在词元位置 $i$，我们将应用一个成对旋转矩阵 $R_i$，得到 $q'^{(i)} = R_i q^{(i)} = R_i W_q x^{(i)}$。这里，$R_i$ 将嵌入元素对 $q^{(i)}_{2k-1:2k}$ 作为 2D 向量旋转角度 $\theta_{i,k} = \Theta^{(2k-2)/d}$，对 $k \in \{1, \ldots, d/2\}$ 和某个常数 $\Theta$。因此，我们可以将 $R^i$ 视为大小为 $d \times d$ 的块对角矩阵，块为 $R_k^i$：

$$R_k^i = \begin{bmatrix} \cos(\theta_{i,k}) & -\sin(\theta_{i,k}) \\ \sin(\theta_{i,k}) & \cos(\theta_{i,k}) \end{bmatrix}.$$

虽然可以构造完整的 $d \times d$ 矩阵，但好的解决方案应利用此矩阵的性质来更高效地实现变换。由于我们只关心给定序列中词元的相对旋转，我们可以在层和不同批次之间重用为 $\cos(\theta_{i,k})$ 和 $\sin(\theta_{i,k})$ 计算的值。如果想优化，可以使用单个 RoPE 模块被所有层引用，它可以在 `init` 期间通过 `self.register_buffer(persistent=False)` 创建预计算的 sin 和 cos 值的 2D 缓冲区，而不是 `nn.Parameter`（因为我们不希望学习这些固定的余弦和正弦值）。对 $q^{(i)}$ 执行的相同旋转过程然后对 $k^{(j)}$ 执行，通过对应的 $R^j$ 旋转。注意，该层没有可学习参数。

**Problem (rope): 实现 RoPE** (2 points)

**可交付成果：** 实现一个 `RotaryPositionalEmbedding` 类，将 RoPE 应用于输入张量。建议使用以下接口：

- `theta: float` RoPE 的 $\Theta$ 值
- `d_k: int` 查询和键向量的维度
- `max_seq_len: int` 将输入的最大序列长度

`def forward(self, x: torch.Tensor, token_positions: torch.Tensor) -> torch.Tensor` 处理形状为 `(..., seq_len, d_k)` 的输入张量，返回相同形状的张量。应容忍具有任意批量维度的 x。应假设词元位置是形状为 `(..., seq_len)` 的张量，指定 x 沿序列维度的词元位置。应使用词元位置来切片（可能是预计算的）cos 和 sin 张量。

#### 3.5.4 缩放点积注意力（Scaled Dot-Product Attention）

我们现在将实现 Vaswani 等人 [2017]（第 3.2.1 节）中描述的缩放点积注意力。作为初步步骤，注意力操作的定义将使用 softmax——一种取非归一化分数向量并将其转换为归一化分布的操作：

$$\text{softmax}(v)_i = \frac{\exp(v_i)}{\sum_{j=1}^{n} \exp(v_j)}.$$

注意，对于大值，$\exp(v_i)$ 可能变为 inf（然后 inf/inf = NaN）。我们可以通过注意到 softmax 操作对所有输入添加任何常数 $c$ 是不变的来避免这个问题。我们可以利用这个属性来提高数值稳定性——通常，我们会从所有元素中减去 $o_i$ 的最大条目，使新的最大条目为 0。你现在将实现 softmax，使用这个技巧来保证数值稳定性。

**Problem (softmax): 实现 softmax** (1 point)

**可交付成果：** 编写一个函数，对张量应用 softmax 操作。你的函数应接受两个参数：一个张量和维度 $i$，并对输入张量的第 $i$ 维应用 softmax。输出张量应与输入张量形状相同，但其第 $i$ 维现在将有一个归一化概率分布。使用从所有元素减去第 $i$ 维最大值的技巧来避免数值稳定性问题。

要测试你的实现，完成 `[adapters.run_softmax]` 并确保通过 `uv run pytest -k test_softmax_matches_pytorch`。

我们现在可以如下数学定义注意力操作：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right) V$$

其中 $Q \in \mathbb{R}^{n \times d_k}$，$K \in \mathbb{R}^{m \times d_k}$，$V \in \mathbb{R}^{m \times d_v}$。这里，Q、K 和 V 都是此操作的输入——注意这些不是可学习参数。如果你想知道为什么这不是 $QK^\top$，请参见第 3.3.1 节。

**掩码（Masking）：** 有时遮蔽注意力操作的输出是方便的。掩码应具有形状 $M \in \{\text{True}, \text{False}\}^{n \times m}$，该布尔矩阵的每一行 $i$ 指示查询 $i$ 应关注哪些键。位置 $(i, j)$ 的 True 值表示查询 $i$ 确实关注键 $j$，False 值表示查询不关注该键。从计算上讲，使用掩码比在子序列上计算注意力要高效得多，我们可以通过取预 softmax 值 $\frac{QK^\top}{\sqrt{d_k}}$ 并在掩码矩阵为 False 的任何条目处添加 $-\infty$ 来实现这一点。

#### 3.5.5 因果多头自注意力（Causal Multi-Head Self-Attention）

我们将实现 Vaswani 等人 [2017] 第 3.2.2 节描述的多头自注意力。回想一下，数学上，应用多头注意力的操作定义如下：

$$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h)$$

对 $\text{head}_i = \text{Attention}(Q_i, K_i, V_i)$，

$Q_i, K_i, V_i$ 是分别对应 Q、K、V 的嵌入维度上大小为 $d_k$ 或 $d_v$ 的第 $i \in \{1, \ldots, h\}$ 个切片。由此我们可以构成多头自注意力操作：

$$\text{MultiHeadSelfAttention}(x) = W_O \text{MultiHead}(W_Q x, W_K x, W_V x)$$

这里，可学习参数为 $W_Q \in \mathbb{R}^{hd_k \times d_{model}}$，$W_K \in \mathbb{R}^{hd_k \times d_{model}}$，$W_V \in \mathbb{R}^{hd_v \times d_{model}}$，$W_O \in \mathbb{R}^{d_{model} \times hd_v}$。当你实现这个时，应该用总共三次矩阵乘法来计算键、值和查询投影。

**因果掩码（Causal masking）。** 你的实现应防止模型关注序列中的未来词元。换句话说，如果模型被给予词元序列 $t_1, \ldots, t_n$，并且我们想要计算前缀 $t_1, \ldots, t_i$（其中 $i < n$）的下一词预测，模型不应该能够访问（关注）位置 $t_{i+1}, \ldots, t_n$ 的词元表示。我们将使用因果注意力掩码（causal attention masking），允许词元 $i$ 关注序列中所有位置 $j \leq i$。你可以使用 `torch.triu` 或广播索引比较来构造此掩码，并应利用你的缩放点积注意力实现已经支持注意力掩码这一事实。

**应用 RoPE。** RoPE 应应用于查询和键向量，但不应用于值向量。此外，头维度应作为批量维度来处理，因为在多头注意力中，注意力是对每个头独立应用的。这意味着对每个头的查询和键向量应用完全相同的 RoPE 旋转。

**资源核算（Resource accounting）。** 能够理解 Transformer 各部分如何消耗计算和内存是很有用的。我们将完成一些基本的"FLOPs 核算"。Transformer 中绝大多数 FLOPS 是矩阵乘法，因此我们的核心方法很简单：

1. 写下 Transformer 前向传播中的所有矩阵乘法。
2. 将每次矩阵乘法转换为所需的 FLOPs。

**规则：** 给定 $A \in \mathbb{R}^{m \times n}$ 和 $B \in \mathbb{R}^{n \times p}$，矩阵乘积 $AB$ 需要 $2mnp$ 个 FLOPs。要看到这一点，注意 $(AB)[i,j] = A[i,:] \cdot B[:,j]$，这个点积需要 $n$ 次加法和 $n$ 次乘法（$2n$ 个 FLOPs）。然后，由于矩阵乘积 $AB$ 有 $m \times p$ 个条目，FLOPs 总数为 $(2n)(mp) = 2mnp$。

现在，在做下一个问题之前，最好先遍历 Transformer 块和 Transformer LM 的每个组件，列出所有矩阵乘法及其相关的 FLOPs 成本。
